## 2. 核心規則

### 2.1 牌組

| 類別 | 牌 | 數量 |
|------|----|------|
| **萬子** | 一萬 ～ 九萬 | 各 4 張，共 36 |
| **筒子** | 一筒 ～ 九筒 | 各 4 張，共 36 |
| **條子** | 一條 ～ 九條 | 各 4 張，共 36 |
| **風牌** | 東 南 西 北 | 各 4 張，共 16 |
| **三元牌** | 中 發 白 | 各 4 張，共 12 |
| **花牌** | 梅蘭竹菊 / 春夏秋冬 | 各 1 張，共 8 |
| **總計** | | **144 張** |

> Pomax 已是 144 張，牌組不需變動。

### 2.2 手牌張數

| 項目 | Pomax 原版 | 台灣版 |
|------|-----------|--------|
| 手牌 | **13 張** | **16 張** |
| 莊家起手 | 14 張 | **17 張**（多摸一張直接打出） |
| 胡牌 | 14 張（4 面子 + 1 雀頭） | **17 張（5 面子 + 1 雀頭）** |

> ⚠️ 這是最大的改動點。所有發牌、摸牌、胡牌判定、AI 邏輯都需要配合調整。

### 2.3 基本流程

```
1. 洗牌 → 砌牌（17 墩 × 2）
2. 莊家擲骰 → 決定開門位置
3. 每人取 16 張（莊家 17 張）
4. 補花（摸到花牌立即攤出，從牌尾補牌）
5. 莊家打出第一張
6. 逆時針輪流：摸牌 → 打牌
7. 其他玩家可吃/碰/槓/胡
8. 直到有人胡牌或流局（荒牌）
```

### 2.4 吃碰槓優先順序

```
胡 > 槓 > 碰 > 吃
```

- **吃**：僅限下家
- **碰**：任何人
- **槓**：任何人（明槓）/ 自己摸到（暗槓 / 加槓）
- **胡**：任何人

### 2.5 一炮多響

> 一人放炮，多家同時宣告胡牌時，**全部都胡**，放炮者分別賠付每一家。

## 3. 計分系統（台數制）

### 3.1 基本結構

```
每家支付 = 底 + (台 × 每台金額)
```

- **底**：固定基本分（預設 300）
- **台**：依牌型累加
- **每台金額**：預設 100
- 可在設定中調整底和台的金額

### 3.2 台數表

#### 基本台

| 牌型 | 台數 | 說明 |
|------|------|------|
| **自摸** | 1 台 | 自己摸到胡牌 |
| **門清** | 1 台 | 沒有吃碰槓（全暗牌） |
| **門清自摸** | 3 台 | 門清 + 自摸（不另計門清1台+自摸1台，直接3台） |
| **平胡** | 2 台 | 全部順子 + 非字牌雀頭，且不能自摸 |
| **碰碰胡** | 4 台 | 全部刻子（碰/槓） |
| **全求人** | 2 台 | 全部吃碰，聽牌靠別人打 |
| **混一色** | 4 台 | 同一花色 + 字牌 |
| **清一色** | 8 台 | 全部同一花色，無字牌 |

#### 字牌台

| 牌型 | 台數 | 說明 |
|------|------|------|
| **三元牌刻子** | 1 台 | 中/發/白 的碰或槓，每組 1 台 |
| **自風刻子** | 1 台 | 自己門風的碰或槓 |
| **圈風刻子** | 1 台 | 當前圈風的碰或槓 |

#### 特殊台

| 牌型 | 台數 | 說明 |
|------|------|------|
| **槓上開花** | 1 台 | 槓後補牌剛好胡 |
| **搶槓胡** | 1 台 | 別人加槓時胡那張牌 |
| **海底撈月** | 1 台 | 摸最後一張牌胡（自摸） |
| **河底撈魚** | 1 台 | 最後一張打出的牌胡 |
| **花牌** | 每張 1 台 | 每張花牌算 1 台 |
| **正花** | 額外 1 台 | 花牌對應自己的門風位置 |
| **一槓** | 1 台 | 有一組槓 |
| **兩槓** | 2 台 | 有兩組槓 |

#### 大牌（限制牌型）

| 牌型 | 台數 | 說明 |
|------|------|------|
| **小三元** | 4 台 | 兩組三元碰/槓 + 一組三元雀頭 |
| **大三元** | 8 台 | 三組三元牌全碰/槓 |
| **小四喜** | 8 台 | 三組風牌碰/槓 + 一組風牌雀頭 |
| **大四喜** | 16 台 | 四組風牌全碰/槓 |
| **字一色** | 16 台 | 全部字牌 |
| **天胡** | 16 台 | 莊家起手直接胡 |
| **地胡** | 16 台 | 閒家第一次摸牌即胡 |

> **備註：** 台數可以累加。例如「門清自摸 + 混一色 + 三元刻」= 3 + 4 + 1 = 8 台。
> 大牌（天地胡、大四喜等）通常不再額外加其他台。

## 4. 莊家規則

### 4.1 起莊
- 遊戲開始時擲骰決定莊家（或預設玩家為東風/莊家）

### 4.2 連莊
- 莊家胡牌 → **連莊**（繼續當莊）
- 莊家連莊時，**加 1 台**（連一拉一：連莊一次多 1 台，兩次多 2 台，以此類推）

### 4.3 過莊（輪莊）
- 莊家沒胡（別人胡或流局且莊家沒聽）→ 莊家換到下家
- 流局時莊家有聽牌 → 連莊

### 4.4 圈風
- 四人各做過一次莊 = 一圈
- 東風圈 → 南風圈 → 西風圈 → 北風圈
- 一般打一圈（東風戰）或半莊（東南）即可，設定中可選

---

## 5. 流局規則

- **荒牌流局**：牌墩摸完無人胡牌
- 流局時依「有聽 / 無聽」決定是否連莊
- 無需計算流局罰分（簡化版）

## 6. AI 行為

| 項目 | 說明 |
|------|------|
| **AI 數量** | 3 個 AI 對手 |
| **難度** | 沿用 Pomax 原版 AI 邏輯，調整為 16 張 |
| **行為** | 吃碰槓判斷、聽牌判斷、防守打牌 |
| **優先級** | 先求胡，有防守意識（看牌河避免放炮） |

> 初版不需要大改 AI，只要確保 16 張的胡牌判定正確即可。後續可迭代 AI 強度。

## 10. 技術方案概述

### 10.1 Fork & 新增規則
```
src/js/core/scoring/
├── ruleset.js           ← 基類（需修改，支援 16 張）
├── chinese-classical.js ← 原版
├── cantonese.js         ← 原版
└── taiwanese.js         ← 【新增】台灣規則
```

### 10.2 主要改動區域

| 模組 | 改動 |
|------|------|
| **發牌 (deal)** | 每人 16 張 → 莊家 17 張 |
| **胡牌判定** | 5 面子 + 1 雀頭（原為 4+1） |
| **計分** | 新增 `taiwanese.js`，台數制邏輯 |
| **遊戲流程** | 連莊、圈風切換、一炮多響 |
| **UI 文字** | 全部改為繁體中文 |
| **設定** | 新增底/台金額設定 |

### 10.3 開發順序建議

```
Phase 1: 基礎建設
  └─ Fork → 調整為 16 張 → 發牌 / 摸牌 / 手牌上限

Phase 2: 規則引擎
  └─ taiwanese.js → 胡牌判定 → 台數計算

Phase 3: 遊戲流程
  └─ 連莊 / 過莊 → 圈風 → 一炮多響 → 流局

Phase 4: 在地化
  └─ UI 繁中翻譯 → 計分結算畫面

Phase 5: 測試 & 修正
  └─ 利用 Pomax 的 node test 架構跑規則測試
```
